## 1. 問題陳述

### 1.1 目標使用者

| 族群 | 規模 | 痛點 |
|---|---|---|
| 台股投資人 | ~300 萬 | 每天要追蹤特定股票討論，卻要手動搜尋 PTT 板 |
| 科技業求職者 | ~10 萬 | 想第一時間看到徵才文章，但 PTT 看板文章流速快 |
| 行銷研究者 | ~3,000 | 需追蹤特定主題討論（產品口碑/品牌監測） |
| 鄉民重度使用者 | ~50 萬 | 想自訂關鍵字接收通知，不用每次登入 PTT |

### 1.2 為什麼不做替代方案

- **手動搜尋 PTT**：每天耗 30 分鐘、易錯過重要文章
- **Google Alerts**：無 PTT 支援、僅網頁
- **現有 PTT 監控工具**：介面老舊、無訂閱制、無 LINE 通知
- **我們的解法**：設定關鍵字 + 板名 → 5 分鐘內看到新文章列表 + LINE 即時推播，NT$99-499/月

---

## 2. 解決方案

### 2.1 核心價值主張

> 「你想追蹤的 PTT 文章，自動送到你面前。」

### 2.2 使用者流程

1. 註冊 + 設定追蹤規則（關鍵字 + 板名 + 通知頻率）
2. 系統每 5 分鐘爬取新文章，匹配規則
3. 命中文章即時推播 LINE / Email / Web
4. Dashboard 看見所有追蹤規則 + 命中歷史
5. 一鍵點擊查看原文

---

## 3. 功能清單

### 3.1 MVP（必做）

- [ ] 關鍵字 + 板名規則設定
- [ ] 5 分鐘自動爬蟲（背景排程）
- [ ] 文章列表（含標題/作者/時間/連結）
- [ ] LINE 通知整合（每則命中即時推播）
- [ ] Email 通知整合
- [ ] 規則 CRUD + 啟用/暫停
- [ ] 命中歷史查詢（30 天內）

### 3.2 v2（加值）

- [ ] 進階條件（推爆數/作者/時間區間）
- [ ] AI 摘要（自動總結文章重點）
- [ ] 多用戶協作（團隊共用規則）
- [ ] 統計分析（熱門文章排行）

### 3.3 明確不做

- 看板全文爬取（只抓標題，遵守 PTT 規範）
- 自動發文/回文（不做 PTT 互動）
- 跨境內容（純台灣 PTT）
- 與其他論壇整合（先 PTT）

## 4. 技術棧

| 層 | 選擇 | 理由 |
|---|---|---|
| 前端 | Next.js + TypeScript | 與既有架構一致 |
| 後端 | Next.js API Routes + FastAPI | 背景排程 |
| 排程 | APScheduler | 每 5 分鐘爬蟲 |
| 資料庫 | Prisma + PostgreSQL | 規則/命中歷史 |
| 通知 | LINE Messaging API + SendGrid | LINE + Email |
| 部署 | Vercel + Railway |

## 6. 風險與決策

| 風險 | 等級 | 緩解 |
|---|---|---|
| PTT 阻擋爬蟲 | 🔴 高 | 模擬正常 User-Agent + Rate limit + 失敗重試 |
| 個資疑慮（使用者追蹤紀錄） | 🟡 低 | 加密儲存 + 30 天自動清除 |
| LINE API 成本（高頻推播） | 🟠 中 | 批次推播 + 用量配額 |
| 法律爭議（PTT 內容引用） | 🟠 中 | 僅引用標題 + 原文連結回 PTT |
